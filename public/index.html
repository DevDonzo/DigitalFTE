A local control room for work that waits.

DigitalFTE turns incoming messages into structured work, drafts the next move, and pauses sensitive actions for your approval. The queue stays in your Obsidian vault, where every decision remains readable.

  1. /Needs_Action New input lands as a readable file.

Five operating loops

Local markdown memory

Human approval before sensitive actions

Five loops keep the system moving.

Each loop has one job. Together they take work from an external signal to an approved, auditable action without hiding the handoffs.

  1. 1.0

    Perceive

    Watchers turn Gmail, WhatsApp, and social activity into markdown tasks.

    input → /Needs_Action
  2. 2.0

    Reason

    The agent reads your handbook and writing style before drafting a plan.

    context → /Plans
  3. 3.0

    Review

    Email, payment, and real-time social actions wait for explicit approval.

    draft → /Pending_Approval
  4. 4.0

    Execute

    Focused adapters carry out approved work across communication and finance tools.

    approval → action
  5. 5.0

    Recover

    Health checks and retry rules surface failures without erasing their history.

    result → /Logs
